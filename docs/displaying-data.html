Exibindo dadosEdit on GitHub

A coisa mais básica que você pode fazer com uma interface de usuário é apresentar alguns dados. React torna mais fácil para exibir dados e mantém automaticamente a interface de up-to-data em que as alterações de dados.

Primeiros passos#

Vejamos um exemplo muito simples. Criar um hello-react.htmlarquivo com o seguinte

<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <title>Hello React</title>
    <script src="https://fb.me/react-0.14.6.js"></script>
    <script src="https://fb.me/react-dom-0.14.6.js"></script>
    <script src="https://cdnjs.cloudflare.com/ajax/libs/babel-core/5.8.23/browser.min.js"></script>
  </head>
  <body>
    <div id="example"></div>
    <script type="text/babel">

      // ** Your code goes here! **

    </script>
  </body>
</html>

Para o resto da documentação, vamos apenas focar o código JavaScript e assumir que é inserido em um modelo como o descrito acima. Substitua o espaço reservado comentário acima com a seguinte JSX:

var HelloWorld = React.createClass({
  render: function() {
    return (
      <p>
        Hello, <input type="text" placeholder="Your name here" />!
        It is {this.props.date.toTimeString()}
      </p>
    );
  }
});

setInterval(function() {
  ReactDOM.render(
    <HelloWorld date={new Date()} />,
    document.getElementById('example')
  );
}, 500);

Reactive Atualizações#

Abra hello-react.htmlem um navegador da web e digite seu nome no campo de texto. Observe que React apenas está mudando a seqüência de tempo na UI - qualquer entrada que você colocou no campo de texto permanece, mesmo que você não tenha escrito qualquer código para gerenciar esse comportamento. React figuras-lo para fora para você e faz a coisa certa.

A forma como somos capazes de descobrir isso é que React não manipular o DOM, a menos que ele precisa. Ele usa, um DOM simulada interno rápido para realizar comparações e calcula a mutação DOM mais eficiente para você.

As entradas para este componente são chamados de props - abreviação de "propriedades". Eles são passadas como atributos na sintaxe JSX. Você deve pensar destes como imutável dentro do componente, ou seja, nunca escrever a this.props.

Os componentes são exatamente como funções#

React componentes são muito simples. Você pode pensar neles como funções simples que levam em props e state(discutido mais tarde) e renderizar HTML. Com isto em mente, os componentes são fáceis de se trabalhar.

Nota:

Uma limitação: React componentes só pode processar um único nó raiz. Se você quiser retornar vários nós eles devem ser acondicionados em uma única raiz.

JSX Sintaxe #

Nós acreditamos fortemente que os componentes são o caminho certo para separar preocupações em vez de "modelos" e "lógica de exibição." Pensamos que a marcação eo código que gera estão intimamente ligados. Além disso, a lógica de exibição é frequentemente muito complexo e utilizando linguagens de modelo para expressá-lo torna-se complicado.

Descobrimos que a melhor solução para este problema é gerar árvores HTML e componentes directamente a partir do código JavaScript, que você pode usar todo o poder expressivo de uma linguagem de programação real para construir UIs.

A fim de tornar isso mais fácil, nós adicionamos uma sintaxe HTML-como muito simples, opcional para criar estes React nós de árvore.

JSX permite criar objetos JavaScript usando a sintaxe HTML para gerar um link em React usando JavaScript puro você escreveria.:

React.createElement('a', {href: 'https://facebook.github.io/react/'}, 'Hello!')

Com JSX este torna-se:

<a href="https://facebook.github.io/react/">Hello!</a>

Nós encontramos isso fez edifício React aplicativos mais fácil e os designers tendem a preferir a sintaxe, mas todo mundo tem seu próprio fluxo de trabalho, de modo JSX não é necessário para usar React.

JSX é muito pequena. Para saber mais sobre isso, consulte JSX in depth. Ou veja o transformar em ação O Babel REPL.

JSX é semelhante ao HTML, mas não exatamente o mesmo. Veja pegadinhas JSX para algumas diferenças fundamentais.

Babel expõe uma série de maneiras de começar a usar JSX, que vão desde ferramentas de linha de comando para Ruby on Rails integrações. Escolha a ferramenta que funciona melhor para você.

React sem JSX #

JSX é completamente opcional; você não tem que usar JSX com React. Você pode criar React elementos em JavaScript simples usando React.createElement o que leva um nome de marcação ou componente, as propriedades de um objeto, e número variável de argumentos criança opcionais.

var child1 = React.createElement('li', null, 'First Text Content');
var child2 = React.createElement('li', null, 'Second Text Content');
var root = React.createElement('ul', { className: 'my-list' }, child1, child2);
ReactDOM.render(root, document.getElementById('example'));

Para maior comodidade, você pode criar funções de fábrica de curto mão para criar elementos de componentes personalizados.

var Factory = React.createFactory(ComponentClass);
...
var root = Factory({ custom: 'prop' });
ReactDOM.render(root, document.getElementById('example'));

React já foi construído com fábricas de tags HTML comuns:

var root = React.DOM.ul({ className: 'my-list' },
             React.DOM.li(null, 'Text Content')
           );
