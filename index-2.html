Apenas a interface do usuÁrio

Muitas pessoas usam React como o V no MVC. React não faz suposições sobre o resto de sua tecnologia de pilha, é fácil para testá-lo em um pequeno recurso em um projeto existente.

DOM Virtual

React abstrai o DOM de você, dando um modelo de programação mais simples e melhor desempenho. React também pode processar no servidor usando Node, e pode acionar aplicativos nativos usando React Nativo

Fluxo de dados

React implementos one-way fluxo de dados reacts que reduz clichê e é mais fácil para raciocinar sobre a ligação de dados tradicionais.

Um componente simples

React componentes implementar uma render() método que usa dados de entrada e retorna o que mostrar. Este exemplo utiliza uma sintaxe XML, chamado JSX. Os dados de entrada que é passado para o componente pode ser acessado por render() via this.props.

JJSX é opcional e não necessário para usar React. Experimente clicar sobre "JS Compilado" para ver o código JavaScript em bruto produzido pelo compilador JSX.

Um componente Stateful

Além de levar os dados de entrada (acessado atravésthis.props um componente pode manter dados de estado internas (acessados ​​via this.state Quando as alterações de dados de estado de um componente, a marcação processada será atualizado por re-invocando render()

 

Uma aplicação

Usando props e state podemos montar um pequeno aplicativo Todo. Este exemplo usa state para acompanhar a atual lista de itens, bem como o texto que o usuário digitou. Apesar de manipuladores de eventos parecem ser prestados em linha, eles serão recolhidos e implementado usando a delegação de evento.

 

Um componente usando plugins externos

React é flexível e fornece ganchos que lhe permitem interagir com outras bibliotecas e frameworks. Este exemplo usa marcada, uma biblioteca Markdown externo, para converter o valor do textarea em tempo real.
